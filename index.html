Project Overview

What is Real-Time Scheduling?

Real-time operating systems (RTOS) must process data as it comes in, typically without buffering delays. Processing time requirements (including any OS delay) are measured in tenths of seconds or shorter.

Why is it Used?

It is crucial for systems where a late result is as bad as a wrong result. Applications include pacemakers, anti-lock brakes, aircraft flight controls, and industrial automation.

Focus of this Project

This simulator compares two pillar algorithms: Rate Monotonic Scheduling (RMS), a static-priority assignment method, and Earliest Deadline First (EDF), a dynamic-priority algorithm.

Analyzed Algorithms

Rate Monotonic Scheduling (RMS)

Static Priority
  • Assigns priorities statically before execution.
  • Shorter period = Higher priority.
  • Simplest and most widely used in practical RTOS.
  • Only guarantees schedulability up to ~69% utilization bound for large sets.

Earliest Deadline First (EDF)

Dynamic Priority
  • Assigns priorities dynamically during execution.
  • Earlier absolute deadline = Higher priority.
  • Theoretically optimal algorithm.
  • Can guarantee schedulability up to 100% CPU utilization.
